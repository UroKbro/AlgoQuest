# ALGO_TECH_STACK.md: The Simplified Logic Engine

To achieve the "simple yet effective" vision, the tech stack is focused on **zero-install, browser-first execution.**

---

## 1. Frontend & Execution (The "Zero-Friction" Core)
- **Framework**: **React.js**. Manages state-heavy time-travel scrubbing.
- **Python Runtime**: **Pyodide** (WebAssembly). Allows Python code to run entirely in the browser.
  - *Benefit*: No backend servers needed for compilation; instant feedback; offline capable.
- **Styling**: **Pure CSS**. Custom variables and keyframes for "Cyber-Minimalist" visuals.

---

## 2. Visualization Engines
- **HTML5 Canvas**: Primary engine for 2D visualizers (Dojo, Lab, Sandbox).
- **React Flow**: For visualizing **World** project architectures.
- **Framer Motion**: For "Spring-Physics" UI transitions and weighted motion.

---

## 3. Intelligence & Feedback Loop
- **Gemini API**:
  - **The Critique Loop**: Analyzes user-submitted code from the Dojo or World and provides iterative feedback.
  - **Socratic Tutor**: Generates guiding questions based on the visualization state.
  - **Neural Bridge**: Converts natural language "What If" ideas into Python syntax.

---

## 4. Backend & Data (Minimalist)
- **FastAPI (Python)**: Acts as a thin metadata layer to serve algorithm definitions and handle user profile saves.
- **SQLite**: Lightweight storage for progress, belts, and "Logic Posters."

---

## 5. Deployment
- **Platform**: Vercel (Edge delivery).
- **Architecture**: **Static Site + Client-Side Compilation**. The bulk of the logic happens on the user's machine, ensuring the lowest latency possible.
